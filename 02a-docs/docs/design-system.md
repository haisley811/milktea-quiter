# 设计系统约定

## 视觉方向

- 最大宽度 430px，保持移动 App 壳感。
- 使用浅紫、奶白、薄荷蓝、粉紫和少量蜂蜜橙。
- 卡片使用半透明奶白、细描边、弱阴影和 `backdrop-filter`。
- 字段标签左对齐，提示文字保持小一号、低对比但可读。
- 统计卡片最小高度 112px，长数字允许换行，不挤压旁边内容。

## 可切换 UI 风格

当前 App 不再只有一套固定外观，`我的` 页面提供 6 套可切换视觉风格。用户点击风格按钮后，页面背景、手机外框、卡片、按钮、底部导航、输入框、弹窗、Toast、选中态、状态色和风格色块都会一起变化。

1. `软糖奶茶`：默认风格，粉紫、薄荷绿、玻璃卡片和柔和阴影，保留最初的陪伴感。
2. `黑白极简`：黑白灰、高留白、细边框和更克制的工具感。
3. `液态玻璃`：通透蓝紫、强模糊、高光层和偏 iOS 的玻璃质感。
4. `千禧甜心`：高饱和粉、青、黄，带 Y2K 网页和贴纸感。
5. `新粗野`：粗黑边、硬阴影、高对比色块，偏潮流海报和活动页。
6. `霓虹赛博`：深色底、荧光绿蓝和高亮描边，夜间个性更强。

风格偏好保存在浏览器本地，key 是 `milkTeaUiPreferences`。小程序迁移时可以先保留这 6 个主题名，再把 token 翻译成小程序 WXSS 变量。

## 人物显示偏好

`我的` 页面提供 `显示人物` 开关。打开时，首页、记录页和数据页展示小人、Body Score、当前阶段和角色进度；关闭时，人物相关 UI 会一起隐藏。

关闭人物后需要隐藏的范围包括：

1. 首页角色图和角色状态区域。
2. 记录页紧凑小角色。
3. 数据页角色图、Body Score 面板、Body Score 仪表和阶段进度。
4. 提交结果弹窗里的 Body Score 反馈。

这个开关只改变视觉和反馈密度，不删除历史记录，也不改变价格、热量、糖分等统计数据。

## 交互标准

- 可点击元素需要 hover、active、focus-visible、selected、disabled 或 busy 状态。
- 页面切换 240ms，底部导航切换光条 220ms，循环光条 620ms。
- 弹窗进入 220ms，Toast 进入 260ms，展示 1800ms。
- 提交反馈 260ms，loading 转圈 780ms，骨架 shimmer 1400ms。
- 实时估算卡片在数据变化时使用 `estimate-pop` 220ms。
- 支持系统“减少动态效果”偏好。

## 页面分工

- 首页：角色、今日状态、2x2 数据卡和主操作。
- 记录：模式、点单字段、实时估算、提交影响提示。
- 数据：状态、Body Score、趋势、角色进度、历史记录、删除记录、重置记录。
- 我的：个人账号、登录状态、同步方式、后续小程序和 iOS 账号数据。

## 迁移 Token

设计 token 在 `lib/designTokens.ts`。迁移到小程序或 iOS 时优先复用颜色、圆角、阴影、动效时长、最大宽度和无障碍状态约定。
